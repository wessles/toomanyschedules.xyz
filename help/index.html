Adding Classes

The class list looks weird: it has a bunch of strange symbols like ".", "$" and "^"" which certainly aren't in class section names.

The class list uses something called regular expressions ("Regex"). Regex is a powerful tool for "pattern-matching". Fortunately, you don't need to know all the intricacies of Regex to use toomanyschedules.xyz, though. Here's what you need to know:

Example:

CS 135.$

The dot means "this can be any character"

CS 135[A-PR-Z]$

Alternatively [..] means "any character in these ranges."

CS 135.$

The dollar means "this is the end of the name"


Note: the class list is stored in your cookies whenever you type, so don't worry about losing your work. You can press "reset" to get the example back.


Executing The Search

The term usually looks like "2019F" and "2019S", but for summer sessions and other terms it can vary. Visit sitscheduler.com for a full list (see the "term" dropdown on the right).

The log may say "reloading catalog" and hang for a bit. If this goes on for ten seconds, just reload the page. The reason for this is chrome reloading the cached version of the catalog file.

Results

The results show when you have found possible schedules. You can see them through sitscheduler.com by clicking on the link. In case you're curious, the numbers listed in each link are the call numbers for each section of the schedule.
